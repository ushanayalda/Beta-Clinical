# Codex runbook

## Current state

```text
Generation: FROZEN
Mapped patterns: 40
Mapped case slots: 160
Clinical cases generated: 0
Study-ready cases: 0
Website renderer: ready
```

The repository contains all project rules needed to begin from one case ID. The user does not prepare an additional case specification.

## 1. Verify the package

From the system root:

```bash
python -m pip install -r requirements.txt
python scripts/validate_package.py
python -m unittest tests/test_system.py -v
python tests/browser_smoke.py
```

## 2. Keep generation frozen until the exact command

Status:

```bash
python scripts/manage_queue.py status
```

Freeze immediately:

```bash
python scripts/manage_queue.py freeze --date YYYY-MM-DD
```

## 3. Start one authorised case

Only after the user says the exact command:

```text
BUILD_CASE CP-P001-C001
```

record it:

```bash
python scripts/manage_queue.py start CP-P001-C001 \
  --by "Named operator" \
  --date YYYY-MM-DD
```

Resolve the repository context:

```bash
python scripts/resolve_case.py CP-P001-C001 \
  --output repo/queue/CP-P001-C001_context.json
```

The resolved context is generated by the repository. It is not completed manually.

## 4. Case Builder handoff

Open the Clinical Pathway Case Builder Custom GPT and send:

```text
BUILD_CASE CP-P001-C001
```

Save its single JSON response as:

```text
repo/cases/draft/CP-P001-C001_v1.0.0_case-draft.json
```

Then run:

```bash
python scripts/compute_hash.py case \
  repo/cases/draft/CP-P001-C001_v1.0.0_case-draft.json \
  --curriculum registries/curriculum.json --write

python scripts/validate_canonical_case.py \
  repo/cases/draft/CP-P001-C001_v1.0.0_case-draft.json \
  --curriculum registries/curriculum.json
```

If validation fails, return exact errors to Case Builder with `REPAIR_CASE CP-P001-C001`.

## 5. Archive sources

The canonical draft contains source URLs. Archive them:

```bash
python scripts/archive_sources.py \
  repo/cases/draft/CP-P001-C001_v1.0.0_case-draft.json \
  --output-dir repo/sources/archive/CP-P001-C001_v1.0.0
```

Validate the archived hashes:

```bash
python scripts/validate_canonical_case.py \
  repo/cases/draft/CP-P001-C001_v1.0.0_case-draft.json \
  --curriculum registries/curriculum.json \
  --source-dir repo/sources/archive/CP-P001-C001_v1.0.0
```

Archiving resets review state because the canonical content hash changes.

## 6. Review and lock the canonical case

Complete independent structure review and named human source, clinical, and natural-timing reviews for the same authoring hash.

Update workflow fields only after the decisions exist. Validate again, then save the immutable locked case:

```text
repo/cases/locked/CP-P001-C001_v1.0.0_case-locked.json
```

Never overwrite it.

## 7. Reasoning Builder handoff

Preserve the exact current registry before the build:

```bash
cp repo/registry/master-reasoning-registry.json \
  repo/registry/archive/master-reasoning-registry_v1.0.0.json
```

Attach to the Reasoning Builder:

```text
locked canonical case JSON
repo/registry/archive/master-reasoning-registry_v1.0.0.json
```

Send:

```text
BUILD_REASONING
```

Save:

```text
repo/reasoning/draft/CP-P001-C001_v1.0.0_reasoning-draft.json
```

Validate:

```bash
python scripts/compute_hash.py reasoning \
  repo/reasoning/draft/CP-P001-C001_v1.0.0_reasoning-draft.json --write

python scripts/validate_reasoning_layer.py \
  repo/reasoning/draft/CP-P001-C001_v1.0.0_reasoning-draft.json \
  --case repo/cases/locked/CP-P001-C001_v1.0.0_case-locked.json \
  --registry repo/registry/archive/master-reasoning-registry_v1.0.0.json
```

## 8. Independent audit handoff

Open a fresh Independent Auditor conversation. Attach:

```text
canonical locked case
reasoning draft
exact build-time registry snapshot
all archived source files
```

Send:

```text
AUDIT_CASE
```

Save:

```text
repo/audits/CP-P001-C001_v1.0.0_audit.json
```

Validate:

```bash
python scripts/validate_audit_report.py \
  repo/audits/CP-P001-C001_v1.0.0_audit.json \
  --case repo/cases/locked/CP-P001-C001_v1.0.0_case-locked.json \
  --reasoning repo/reasoning/draft/CP-P001-C001_v1.0.0_reasoning-draft.json \
  --registry repo/registry/archive/master-reasoning-registry_v1.0.0.json \
  --source-dir repo/sources/archive/CP-P001-C001_v1.0.0
```

Open findings return to the responsible generator. Re-audit after every repair.

## 9. Human reasoning review and registry merge

After independent audit, complete named human clinical-safety, educational-load, candidate-language, and final-content reviews for the exact reasoning hash.

Save the approved artifact in:

```text
repo/reasoning/approved/
```

Extract and review the provisional patch:

```bash
python scripts/extract_registry_patch.py \
  --reasoning repo/reasoning/approved/CP-P001-C001_v1.0.0_reasoning-approved.json \
  --output repo/reasoning/approved/CP-P001-C001_v1.0.0_registry-patch.json
```

Merge only after named human approval:

```bash
python scripts/merge_registry_patch.py \
  --registry repo/registry/master-reasoning-registry.json \
  --patch repo/reasoning/approved/CP-P001-C001_v1.0.0_registry-patch.json \
  --new-version 1.1.0 \
  --output repo/registry/master-reasoning-registry.next.json
```

Review the diff before replacing the current registry.

## 10. Export the website case

```bash
python scripts/export_learner.py \
  --case repo/cases/locked/CP-P001-C001_v1.0.0_case-locked.json \
  --reasoning repo/reasoning/approved/CP-P001-C001_v1.0.0_reasoning-approved.json \
  --registry repo/registry/archive/master-reasoning-registry_v1.0.0.json \
  --audit repo/audits/CP-P001-C001_v1.0.0_audit.json \
  --source-dir repo/sources/archive/CP-P001-C001_v1.0.0 \
  --release-date YYYY-MM-DD \
  --output repo/website-data/CP-P001-C001_v1.0.0_learner.json

python scripts/build_index.py \
  --input-dir repo/website-data \
  --output repo/website-data/index.json
```

Copy only learner files to the static site:

```bash
cp repo/website-data/*.json website/data/
```

Run:

```bash
python tests/browser_smoke.py
```

## 11. Update queue paths

```bash
python scripts/manage_queue.py set-stage CP-P001-C001 study_ready \
  --path-field learner_path \
  --path repo/website-data/CP-P001-C001_v1.0.0_learner.json \
  --date YYYY-MM-DD
```

## Automation boundary

This package does not call Custom GPTs programmatically. Their JSON outputs enter the repository through the handoff above. Codex then performs deterministic work.

An API adapter may be added later. It must preserve the same schemas, hashes, source gates, human reviews, and repository boundaries.
